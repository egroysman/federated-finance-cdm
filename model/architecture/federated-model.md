## Federated Data Model

This model brings together data from multiple domains:

- ERP (Finance – invoices, payments)
- CRM (Customer and sales data)
- Operational systems

Instead of duplicating logic across systems, data is standardized into a Common Data Model (CDM) where:

- Entities are aligned across domains
- Transformations are centralized
- Data is reusable across reporting and AI use cases

This approach enables consistent, scalable analytics without rebuilding pipelines for each use case.
